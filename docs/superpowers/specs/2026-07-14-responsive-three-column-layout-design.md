# AI Studio OS 自适应三栏窗口设计规格

## 状态

- 方案：A，自适应三栏
- 用户确认：2026-07-14
- 目标页面：Figma 文件 `pw09dkGSbYLOhRNFzLvuHP`，节点 `26:773`
- 目标平台：Windows Tauri 桌面应用

## 背景与根因

当前首页先按 `1440 × 1024` 固定画布绝对定位，再由 `DesignViewport` 对整张画布执行 `translate(...) scale(...)`。缩放比例采用覆盖容器的算法，因此窗口宽高比偏离设计稿时，画布会超出窗口并被裁切。用户看到的效果等同于在一个可缩放的相机窗口中观看固定图片：窗口尺寸变化了，但组件本身没有重新排版。

圆角白边来自窗口透明层、页面背景和多层圆角裁切不一致。根容器未始终以同一深色背景覆盖完整内容区时，透明窗口背后的系统或宿主底色会从圆角和缩放空隙中露出。

本规格取代 `2026-07-14-desktop-window-responsive-design.md` 中“整张 Figma 画布等比缩放”的方案。Figma 默认尺寸下的视觉基准继续有效，但非默认尺寸改为真实响应式布局。

## 目标

1. 在 `1440 × 1024` 内容区保持 Figma 节点 `26:773` 的 `240 / 860 / 340` 三栏几何和视觉表现。
2. 窗口宽度变化时，让侧栏、中心工作区、任务中心真实改变尺寸和位置，不再整体缩放或裁切。
3. 在最小内容区 `1024 × 720` 下，三块主要区域仍可访问、文字与控件保持可读。
4. 窗口高度不足时，中心工作区和任务中心分别纵向滚动，不缩小字号、按钮或卡片。
5. 根背景始终完整覆盖窗口，非最大化圆角无白边，最大化状态无边缘缝隙。

## 不在范围

- 不改变 Figma 的颜色、字体、图标、图片、文案和默认尺寸下的视觉层级。
- 不新增任务抽屉或把任务中心移动到页面下方。
- 不实现小于 `1024 × 720` 的额外布局；原生窗口最小尺寸负责阻止进入该范围。
- 不新增后端、AI 服务、登录、支付或持久化业务。
- 不用截图背景、自制图标或替代资源取代 Figma 导出的素材。

## 采用方案

### 1. 根布局

页面根结构使用真实 CSS Grid，而不是固定画布与变换：

```text
window-content
├── sidebar
├── workspace-scroll-region
└── task-scroll-region
```

- `html`、`body`、`#root`、窗口内容根节点全部使用 `width: 100%`、`height: 100%` 和统一深色背景。
- 移除根级 `transform: scale(...)`、`translate(...)`、固定 `1440px × 1024px` 视口和居中偏移计算。
- 页面根节点使用 `min-width: 0`、`min-height: 0`，允许 Grid 子区域正确收缩和滚动。
- 窗口控制按钮位于统一窗口层，所有功能页面始终可见且不参与业务区域滚动。

### 2. 宽度断点

#### 宽屏：`width >= 1280px`

```css
grid-template-columns: 240px minmax(0, 1fr) clamp(320px, 23.611vw, 340px);
```

- 左侧栏固定 `240px`，显示图标和文字。
- 中心工作区占据剩余宽度；在 `1440px` 内容宽度下必须精确为 `860px`。
- 任务中心宽度在 `320–340px` 之间；在 `1440px` 下必须精确为 `340px`。
- 首页内部关键节点在 `1440 × 1024` 下继续按 Figma 测量值定位；其他宽度使用 Grid、Flex 和 `minmax()` 分配空间。

#### 紧凑宽度：`1024px <= width < 1280px`

```css
grid-template-columns: 72px minmax(0, 1fr) 280px;
```

- 左侧栏折叠为 `72px` 图标轨道，隐藏导航文字、品牌文字和非必要数值说明。
- 每个图标保留原有功能、选中态、悬停态和可访问名称；悬停时可显示文字提示。
- 任务中心保持约 `280px`，始终位于右侧，不改为抽屉。
- 中心工作区获得剩余空间，在最小窗口下约为 `672px`。
- 中心区顶部快捷入口允许由四列变为两列。
- “最近创作 / 当前项目”等卡片在空间不足时允许由双列变为单列，但内容顺序和信息层级不变。

#### 小于最小宽度

- Tauri 内容区设置 `minWidth: 1024`，不支持继续缩小。
- 浏览器开发预览若被强制缩到更窄，只允许出现区域内部滚动，不恢复整页缩放。

### 3. 高度与滚动

- 页面根布局固定占满可用内容高度，不让整个固定画布向窗口外溢出。
- 左侧栏保持完整高度，账户区固定在栏底；导航区在极端内容溢出时可独立滚动。
- 中心工作区使用 `overflow-y: auto`，顶部标题、输入区、快捷入口和下方卡片按文档流排列。
- 任务中心使用独立 `overflow-y: auto`，任务列表和积分卡都可到达。
- 两个滚动区互不影响；滚动任务中心时，中心创作区保持当前位置，反之亦然。
- 禁止为适应高度而缩小字号、图标、按钮、进度环或卡片。

### 4. 默认尺寸的 Figma 还原

- `1440 × 1024` 是像素级视觉验收基准。
- 三栏边界必须位于 `x = 240` 和 `x = 1100`。
- 现有 Figma 图标、头像、项目封面和最近创作图片继续使用 `public/figma/dashboard/` 中的本地资源。
- 绝对定位只允许保留在局部、尺寸稳定的装饰层中；主三栏、卡片组和内容区不得依赖跨页面绝对坐标。
- 默认尺寸的字体、间距、圆角、边框、颜色和图片裁切继续以 Figma 截图为唯一视觉事实来源。

## 窗口圆角与背景

- 非最大化状态只在最外层 `window-content` 应用一次 `14px` 圆角和 `overflow: hidden`。
- 业务页面根节点不再重复窗口级圆角，也不设置透明外边距。
- `html`、`body`、`#root`、`window-content` 使用相同的深色背景，消除透明层露出的白底。
- 最大化状态将最外层圆角设为 `0`，内容贴合屏幕边缘。
- Tauri 保持无边框窗口；透明能力仅用于外层窗口形状，不允许业务内容背景透明。
- 不通过额外白色或半透明边框模拟系统窗口阴影。

## 组件职责

### `DesignViewport`

- 移除尺寸测量、缩放比例和居中偏移职责。
- 若仍保留组件名，只作为填满父级的结构容器；更推荐将其替换为无变换的 `ResponsiveShell`。

### `Dashboard`

- 提供三栏语义结构和首页数据。
- 用 CSS Grid/Flex 管理主要模块，不再依赖 `1440 × 1024` 根画布的跨区绝对坐标。
- 根据媒体查询切换完整侧栏和图标轨道表现。

### `WindowChrome`

- 统一承载拖动区域、最小化、最大化/还原和关闭按钮。
- 监听最大化状态，只控制外层圆角和窗口按钮状态。
- 不参与业务页面缩放，不隐藏任一功能页面的窗口按钮。

### 其他功能页面

- AI 对话、图片等页面继续通过同一个窗口外框展示。
- 公共根容器不得重新引入固定画布缩放。
- 若页面本身仍有固定布局，应按相同原则逐步改为自身的响应式区域；本次至少保证窗口外框、背景和按钮一致。

## 状态与错误处理

- 浏览器环境没有 Tauri API 时，窗口按钮保持可见但安全跳过桌面命令，页面布局仍可预览。
- 媒体查询只依赖可用内容宽度，不依赖一次性读取的启动窗口尺寸，拖动窗口时自动重新计算。
- 图片加载失败时使用原生替代文本或空资源位，不引入与设计稿不一致的占位图片。
- 滚动条使用深色、低对比样式，不能覆盖主要内容或改变三栏宽度。

## 测试策略

### 纯布局和样式测试

- 断言不再存在根级 `scale(...)`、`translate(...)` 或 `1440 × 1024` 固定画布变换。
- 断言 `1440px` 下三栏为 `240 / 860 / 340`。
- 断言 `1024px` 下三栏为 `72 / 672 / 280`。
- 断言 `1024–1279px` 时导航文字隐藏、图标仍存在且具有可访问名称。
- 断言中心区和任务区都具有独立纵向滚动能力。
- 断言 Tauri 配置包含 `minWidth: 1024` 和 `minHeight: 720`。
- 断言所有页面仍渲染最小化、最大化/还原和关闭按钮。

### 视觉 QA

1. 以 Figma 节点 `26:773` 的 `1440 × 1024` 截图为参考。
2. 在 `1440 × 1024` 截取应用首页，与参考图并排比较三栏边界、关键坐标、字体、间距、圆角和资源。
3. 在 `1280 × 800`、`1180 × 720`、`1024 × 720` 分别截图，检查布局真实重排且没有横向裁切。
4. 在短高度窗口中分别滚动中心区和任务中心，确认两者互不联动且所有内容可达。
5. 检查非最大化窗口四角无白边；最大化时无圆角缝隙。
6. 连续拖动调整窗口尺寸，确认没有整体缩放跳变、空白区域或内容突然消失。

## 验收标准

- 默认窗口与 Figma 节点 `26:773` 保持 1:1 视觉基准。
- 从 `1440 × 1024` 缩放到 `1024 × 720` 时，UI 重新排版而不是整体缩小或裁切。
- 左侧栏在紧凑宽度折叠为图标栏；中心工作区和任务中心始终可见。
- 高度不足时中心工作区和任务中心分别滚动，核心内容均可到达。
- 窗口四周和圆角无白色底边；最大化状态无边缘空隙。
- 所有功能页面都有统一的最小化、最大化/还原和关闭按钮。
- 自动化测试、类型检查、前端构建和 Tauri 构建均通过。
