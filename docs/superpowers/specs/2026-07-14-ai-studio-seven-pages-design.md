# AI Studio OS 七页面统一还原设计规格

## 目标

在现有 React + Tauri 桌面应用中，按 Figma 原始节点 1:1 还原七个 1440 × 1024 页面，并让它们共享一致的窗口框架、侧栏导航、字体层级、图标资源与响应式行为。

## 统一架构

采用已确认的方案 A：共享 `AppShell`，各功能作为独立页面工作区。

- `WindowChrome` 只存在一份，负责拖动、最小化、最大化、关闭、圆角裁切和透明窗口底色。
- `AppSidebar` 只存在一份，负责品牌、主导航、账户卡片和当前页面状态。
- 各页面只负责 Figma 中侧栏右侧的内容区域，不重复实现品牌或窗口控制。
- 1440px 设计宽度下严格保持 240px 侧栏与设计稿几何关系。
- 1024–1279px 使用 72px 图标栏，内容区域使用最小宽度、网格重排和独立滚动，不缩放整张画布，也不留下空白底层。

## 字体规范

所有页面统一使用 Inter、PingFang SC、Microsoft YaHei 的字体回退链，并集中使用以下语义令牌：

- 品牌：17px / 600 / 24px
- 导航与控件：14px / 500 / 20px
- 页面标题：20px / 600 / 28px
- 区块标题：16px / 600 / 24px
- 正文：14px / 400 / 22px
- 辅助文字：12px / 400 / 18px
- 仪表盘主标题是 Figma 指定的例外：32px / 600 / 44px

页面不能再为相同语义重复硬编码不同字号。视觉差异只允许来自 Figma 明确存在的标题层级。

## 视觉与资源规则

- 使用 Figma 返回的真实图片、头像、封面和图标资源。
- 不使用 emoji、字符图标、CSS 手绘图标、占位框或近似渐变代替可见资产。
- 颜色、边框、圆角、阴影和间距由共享令牌管理；页面级样式只描述布局和局部状态。
- 页面背景覆盖整个窗口内容，窗口圆角由最外层深色表面裁切，透明宿主不能透出白边。

## 页面行为

- 侧栏七个入口均可切换到对应页面，并保持单一窗口控制栏。
- AI 对话的新建会话、历史选择、输入框和快捷操作有可见状态。
- AI 图片、视频、音乐的模式、比例、数量、标签、时长和列表选择有可见状态；主按钮可触发完成状态。
- 会员方案可选中，升级、充值按钮有反馈状态。
- 项目页的标签、搜索、排序、网格/列表切换和新建按钮可操作。
- 控件外的次要装饰允许静态，但核心创作路径必须可交互。

## 验收标准

1. 七个导航入口均渲染正确的 `data-node-id`。
2. 1440 × 1024 下逐页截图与对应 Figma 原图同屏比较，修正明显的字号、间距、裁切、边框和圆角偏差。
3. 1024 × 720、1280 × 800、1440 × 1024 下无整页缩放、无白色空白区域、无白色圆角底边。
4. 所有页面都有同一组最小化、最大化、关闭按钮。
5. 单元测试、类型检查、Vite 构建和 Tauri 打包全部通过。
