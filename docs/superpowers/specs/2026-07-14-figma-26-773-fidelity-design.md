# AI Studio OS Figma 26:773 像素级还原规格

## 目标

在现有 React、TypeScript、Vite 与 Tauri 项目中，对默认展示的“创作工作台”首页做像素级校准，使其在 `1440 × 1024` 视口下忠实还原 Figma 文件 `pw09dkGSbYLOhRNFzLvuHP` 的节点 `26:773`。设计稿截图、布局数据与原始图标和图片资源是唯一视觉事实来源。

## 范围

- 校准首页的 240 / 860 / 340 三栏结构、绝对坐标、间距、尺寸、圆角、边框、阴影、颜色和字体。
- 校准左侧导航与账户卡、中央创作区与项目卡、右侧任务中心与积分卡。
- 使用 Figma 节点提供的 SVG、头像、项目封面和最近创作缩略图，不引入图标包或占位资源。
- 保留现有 AI 对话、图片生成等页面及跳转关系，不修改它们的视觉和交互。
- 保留现有 Tauri 窗口能力；浏览器预览仍以 Figma 画布本身为视觉验收对象。
- 保留首页当前的轻量交互，包括导航、AI 对话入口、可点击按钮、输入框及基本焦点和悬停反馈。

## 不在范围

- 真实 AI 生成、服务端接口、登录、支付、持久化、文件上传和项目管理业务。
- 重构与首页还原无关的聊天、图片生成或其他模块。
- 以整张设计稿截图作为页面背景，或用自制图形替代 Figma 原始资源。
- 为非 `1440 × 1024` 视口重新设计响应式布局；较小视口可保持画布裁切或缩放策略，不改变设计比例。

## 实现方案

采用现有实现精修，不重写页面。`src/App.tsx` 继续承载首页组件和现有路由状态，`src/styles/app.css` 负责节点级尺寸与坐标校准，`src/styles/tokens.css` 仅保留可复用的颜色、字体和间距变量。资源继续存放在 `public/figma/dashboard/`，组件通过稳定的本地路径引用。

首页画布固定为 `1440 × 1024`。左侧导航宽 240 像素，中央区域从 x=240 开始、宽 860 像素，任务栏从 x=1100 开始、宽 340 像素。主要模块按 Figma 节点坐标定位，避免 flex 自动分配造成累计偏差；模块内部仍使用 flex 或 grid 组织图标、文案和进度条。

## 组件职责

- `Dashboard`：首页画布、三栏布局和首页导航状态。
- `Task`：任务中心单行任务、进度和对应 Figma 图标。
- 现有聊天与图片生成组件：保持原样，仅通过既有回调与首页连接。
- CSS：首页节点坐标、视觉 token、资源裁切、交互态和视口约束。

组件间仅传递页面跳转回调和展示数据，不新增全局状态、网络请求或业务模型。资源加载失败时保留原生图片替代文本或空图标区域，不使用通用占位图替换设计资源。

## 视觉验收

1. 从 Figma MCP 重新获取节点 `26:773` 的设计上下文和 `1440 × 1024` 截图。
2. 在 `1440 × 1024` 浏览器视口运行首页并截图。
3. 对照全页及左侧栏、创作输入区、最近创作、当前项目、任务中心、积分卡等重点区域。
4. 修复所有 P0、P1 和 P2 视觉偏差，重复截图比较，直到 `design-qa.md` 的最终结论为 `passed`。
5. 字体抗锯齿等平台级差异只在不影响结构和可读性时列为 P3，不为其破坏设计尺寸。

## 测试与验证

- 组件测试验证默认首页、核心中文文案、Figma 资源路径和 AI 对话跳转。
- 样式测试验证 `1440 × 1024` 画布、三栏边界和关键模块坐标。
- 运行 `npm test`、`npm run typecheck` 和 `npm run build`。
- 浏览器检查首页无控制台错误，主要按钮、输入和页面跳转可用。
- 最终 `design-qa.md` 必须存在并写明 `final result: passed`；若浏览器或截图能力不可用，则明确标记为 `blocked`，不宣称完成。

## 成功标准

- 默认首页在 `1440 × 1024` 下与 Figma 节点 `26:773` 的结构、尺寸、内容和资源一致。
- 所有可见图标和图片均来自设计稿导出的本地资源。
- 其他现有页面和跳转没有回归。
- 自动化测试、类型检查和生产构建全部通过。
- 设计 QA 通过，或在外部工具确实不可用时有明确、可复现的阻塞记录。
